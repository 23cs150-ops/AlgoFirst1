DEPARTMENT OF COMPUTER SCIENCE AND ENGINEERING

AlgoFirst: Full-Stack Coding Practice and Evaluation Platform

Objective

  1. Build an interactive coding platform where students solve algorithmic problems inside a browser-based editor.
  2. Execute user code securely with Judge0 and provide instant, test-case-based verdicts.
  3. Persist users, problems, and submissions in MongoDB to support reliable progress tracking.
  4. Enable guided practice through problem filtering, difficulty labeling, and student learning plans.
  5. Provide profile and progress analytics for continuous performance improvement.

Architecture

1) Client Layer (Next.js + React + TypeScript + Tailwind)

Pages: Home, Auth, Problem List, Problem Detail, Progress, Profile, Student Plan
Key UI: Monaco Editor, Problem Statement, Test Results Panel
State: AuthContext and SubmissionContext
HTTP/JSON API (Axios) ↓

2) Application Layer (Express.js)

Route: POST /api/execute
Controller: executeController
Services: judge0Service, db, seedProblems
Responsibility: request validation, execution orchestration, output normalization, verdict generation
Execution Request ↓

3) Execution Layer (Judge0 API)

Compiles and executes submitted code in supported languages.
Returns output, runtime status, and error details to backend.
Store and Query ↓

4) Data Layer (MongoDB + Mongoose)

Collections/Models: User, Problem, Submission
Stores attempts, verdict history, and user progress metrics.

Execution Workflow

  1. User selects a problem and writes code in Monaco Editor.
  2. Frontend sends source code, language, and test inputs to backend API.
  3. Backend forwards code execution request to Judge0.
  4. Judge0 compiles/runs code and returns raw output with execution status.
  5. Backend compares actual output with expected output using normalization logic.
  6. Final verdict and per-test results are returned to frontend UI.
  7. Submission data is saved in MongoDB and reflected in Progress/Profile dashboards.
